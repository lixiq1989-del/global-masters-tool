# AI 全球商科硕士选校工具 — 产品需求文档 (PRD)

> 最后更新: 2026-03-14

## 1. 产品定位

面向中国学生的商科硕士选校决策工具。通过算法智能匹配冲刺/匹配/保底院校，同时提供自由探索、学校画像、项目对比、申请规划等功能，帮助学生高效完成选校决策。

**目标用户**: 计划申请海外商科硕士的中国本科生/在职人士
**核心价值**: 数据驱动选校，替代传统中介的经验判断

## 2. 数据资产

| 数据 | 数量 | 来源 | 文件 |
|------|------|------|------|
| 院校 | 117 所 | 6 国（英美港新澳法） | programs.json |
| 项目 | 1,540 个 | 官网/排名 | programs.json |
| 录取案例 | 24,293 条 | WSO/GMAT Club/OfferCall/飞书模板 | cases.json |
| 就业数据 | 1,599 条 | FT/学校报告/行业报告 | employment.json |

### 数据原则
- 每条录取案例 = 1个真实申请人，不做聚合
- 就业数据以项目为单位，包含均薪/就业率/目标行业/雇主
- 项目数据包含品牌分(1-10)、录取难度分(1-10)、地理分(1-10)
- 所有分数基于公开排名和行业口碑，可追溯

## 3. 功能模块

### 3.1 智能推荐 [已完成] — `/`

**核心逻辑**: 用户填写背景 → 算法匹配 → 输出冲刺/匹配/保底列表

#### 用户输入
- 本科院校（支持国内985/211/双非/海本自动识别）
- GPA（百分制/4.0制自动转换）
- 语言成绩（IELTS/TOEFL）
- GMAT/GRE（可选）
- 实习经历（段数）
- 工作经验（年数）
- 意向专业方向（多选：金融/管理/市场/数据等）
- 目标国家（多选）
- 预算（英镑/年）
- 求职目标城市
- 职业方向

#### 推荐算法 (PATTERN_TABLE)
- **用户强度计算**: tier×GPA 查表得 P50 基准分 + GMAT/WE/语言修正
- **冲刺/匹配/保底分类**: 基于 P25/P75 区间与项目品牌分对比
- **综合匹配分**: admissionFit(40%) + career(25%) + brand(20%) + location(15%)
- **输出**: 每个档位(冲刺/匹配/保底)最多 6 个项目，附推荐理由 + 匹配百分比

#### 结果展示 (ProgramCard)
- 匹配度百分比进度条
- 职业匹配 + 地理匹配指示
- 品牌分/难度/地理分仪表盘
- 学费/均薪/就业率
- 推荐理由列表
- 真实录取案例（最多2条）
- 收藏功能（LocalStorage）
- 链接到探索页/学校页

### 3.2 项目探索 [已完成] — `/explore`

**定位**: 自由浏览全部 1540 个项目的数据库

#### 筛选维度
- 国家/地区（6国）
- 学校（按国家联动）
- 专业方向
- 城市（按国家联动）
- 免GMAT（勾选）
- 无需工作经验（勾选）
- 关键词搜索（学校/项目/方向/城市）

#### 排序选项
- 品牌分↓
- 难度↓
- 学费↑/↓
- 学校名A-Z

#### 交互
- 分页加载（每页20个）
- 统计栏：项目数 + 学校数
- URL参数 `?school=` 支持从其他页面跳转并预设筛选
- 重置按钮

---

### 3.3 学校详情页 [已完成] — `/school/[name]`

**定位**: 单个学校的全景数据画像

#### 学校头部
- 国旗 + 校名 + 层级 + 国家 + 城市
- 快速统计：项目数、专业方向数、录取案例数、就业数据条数
- 链接到探索页筛选该校

#### 项目列表
- 该校全部项目，附品牌分/难度分/学费/均薪
- 标签：专业方向、学制、GMAT要求、工作经验要求

#### 录取背景画像（≥5条案例时显示）
- **数据源**: 中国学生录取案例
- **本科院校层级分布**: 985/211/双非/海本等占比条形图
- **GPA分布**: 分段统计（90%+/85-90%/80-85%/75-80%/<75%）
- **语言成绩分布**: IELTS/TOEFL分段
- **专业背景分布**: Top 8专业
- **GMAT/GRE**: 提交率统计

#### 就业统计
- 平均薪资（£均值）
- 就业率
- 目标行业分布（条形图）
- 目标岗位分布（条形图）
- 主要雇主标签云

### 3.4 项目对比 [待开发] — `/compare`

**定位**: 2-4个项目横向对比

#### 功能设计
- 从收藏列表或搜索中选择项目加入对比
- 对比维度：
  - 基础信息（学校/项目名/国家/城市/学制）
  - 录取要求（GMAT/IELTS/GPA/工作经验）
  - 费用（学费/生活成本估算）
  - 品牌分/难度分/地理分
  - 就业数据（均薪/就业率/目标行业）
  - 录取数据（案例数/录取率统计）
- 差异高亮

#### 数据依赖
- programs.json（项目信息）
- employment.json（就业数据）
- cases.json（案例统计）
- 无需新数据

### 3.5 申请规划报告 [待开发] — `/report`

**定位**: 基于推荐结果生成结构化申请规划

#### 报告内容
1. **背景评估**: 用户强度分析 + 子维度雷达图
2. **选校方案**: 冲刺/匹配/保底各2-3所，附理由
3. **申请时间线**: 根据入学年份倒推关键节点
4. **材料清单**: 各校要求汇总
5. **职业路径**: 基于就业数据的预期分析

#### 技术方案
- 方案A: 纯前端生成（基于推荐结果模板化输出）
- 方案B: 调用 LLM API 生成个性化分析文字
- 输出格式: 页面展示 + PDF导出

## 5. 技术架构

| 层级 | 选型 | 说明 |
|------|------|------|
| 框架 | Next.js 16 (App Router) | Turbopack |
| 语言 | TypeScript | 严格模式 |
| 样式 | Tailwind CSS | 无UI库依赖 |
| 数据 | JSON 静态导入 | 无数据库 |
| 渲染 | 客户端渲染 (use client) | 所有页面 |
| 状态 | React useState/useMemo | 无状态管理库 |
| 存储 | LocalStorage | 收藏功能 |
| 部署 | Vercel（计划） | — |

### 关键设计决策
1. **无数据库**: 全部数据作为JSON打包，客户端处理。适合当前数据规模(~25K条)
2. **纯客户端**: 避免服务端复杂度，便于静态部署
3. **中文UI**: 目标用户为中国学生
4. **模块化组件**: NavBar/ProgramCard/InputForm/ResultSection 独立组件
